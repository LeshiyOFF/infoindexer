# Аудит сбора контактов и поиска по соцсетям

## Архитектура

- **Триггер:** POST `/api/organizations/[id]/contacts` → Redis pub `contacts:parse` → `contacts-parser` (Playwright)
- **Этапы:** Checko, DuckDuckGo, сайт компании, затем поиск по ФИО директора (VK, LinkedIn, Telegram и др.)

## Почему поиск по соцсетям (VK, LinkedIn, Telegram) почти не даёт результатов

### Текущая логика

Для VK, LinkedIn, Telegram и др. используется **один и тот же механизм** `runDirectorSource`:

1. Открывается DuckDuckGo HTML: `https://html.duckduckgo.com/html/?q=site:vk.com "Иванов Иван"`
2. Берётся `page.innerText('body')` — текст страницы DDG
3. Из текста извлекаются email и телефон через `EMAIL_REGEX` и `PHONE_REGEX`

### Ограничение

DuckDuckGo показывает **только сниппеты** выдачи (ссылки и короткие фрагменты). Email и телефон обычно:

- не публикуются в сниппетах VK/LinkedIn;
- скрыты на самих страницах профилей (нужна авторизация, защита от ботов);
- встречаются только на некоторых сайтах (HH.ru, суды, реестры и т.п.).

Поэтому парсинг **страницы DDG**, а не страниц VK/LinkedIn, по определению не находит контакты в соцсетях.

### Что в реальности работает

- **Checko** — прямые контакты на странице компании
- **DuckDuckGo по компании** — контакты в сниппетах (сайты, иногда email/телефон)
- **Официальный сайт** — парсинг страницы компании
- **ЕФРСБ, Росаккредитация** — реестры с контактами
- **HH.ru, госзакупки, суды** — иногда есть контакты в сниппетах DDG

### Варианты улучшения поиска по соцсетям

1. **Переход на целевые страницы** — открывать vk.com/id123, career.habr.com и т.д. Риски: блокировки, капча, политики сайтов.
2. **Использование API** — VK API, LinkedIn API (ограничения, платные тарифы).
3. **Дополнительные поисковые запросы** — изменить запросы DDG, чтобы чаще попадать на страницы с email/телефоном в сниппетах.

## Как проверить сбор контактов

1. Выбрать организацию с директором и выручкой (не однодневка).
2. В админке: `/admin/organizations/[inn]` → кнопка «Запустить парсинг контактов».
3. Смотреть логи: `docker logs infoindexer-contacts-parser-1 -f`.
4. В логах будут этапы: `[Waterfall] Checko...`, `[Waterfall] DuckDuckGo...`, `[Waterfall] VK: ФИО...` и т.д.
5. Для VK/LinkedIn/Telegram `addSource(s.label, emails.length > 0 || phones.length > 0)` — при отсутствии email/телефона в сниппетах DDG всегда будет `found: false`.

## Рекомендации

- Увеличить таймаут для DuckDuckGo (сейчас 15 сек).
- Добавить больше источников с открытыми контактами (2GIS, ФИАС, др.).
- При отсутствии директора в БД — добавить fallback на Checko для получения ФИО.
