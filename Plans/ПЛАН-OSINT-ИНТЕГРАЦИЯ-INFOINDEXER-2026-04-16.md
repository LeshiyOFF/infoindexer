## ОПИСАНИЕ ДОКУМЕНТА

Данный план классифицирует технологии OSINT по степени их применимости к проекту INFOINDEXER с учётом специфики российского рынка.

**Принципы классификации:**

| Категория | Критерий |
|-----------|----------|
| ВНЕДРИТЬ ОДНОЗНАЧНО | Технология даёт уникальную ценность для РФ-рынка |
| СПОРНО | Технология полезна, но есть ограничения/альтернативы |
| НЕ ВНЕДРЯТЬ | Не применима к контексту РФ или дублирует существующее |
| УЖЕ РЕАЛИЗОВАНО | Функциональность полностью работает |
| НУЖНА ДОРАБОТКА | Частично реализовано, требуются улучшения |

# 1. ВНЕДРИТЬ ОДНОЗНАЧНО

Технологии из этой категории дают максимальную ценность для анализа российских компаний.

---

## 1.1. Embeddings / Семантический поиск

### Концепция

Векторный поиск по описанию деятельности компании.

### Что добавит система

- Поиск по описанию ("строительная компания СПб")
- Рекомендации похожих компаний
- Кластеризация

### Сложность внедрения

**Выше средней.** 2-3 недели.

### Аргументация

**Вердикт:** Внедрять поэтапно. YandexGPT > OpenAI для русского.

---

## 1.2. Проверка email на утечки (haveibeenpwned)

### Концепция

Проверка email-адресов директора на наличие в базах утечек паролей.

### Что добавит система

- Сигнал риска кибербезопасности
- Понимание осведомлённости директора

### Технические hint'ы

```typescript
async function checkEmailBreach(email: string): Promise<boolean> {
  const resp = await fetch(`https://haveibeenpwned.com/api/v3/breachedaccount/${email}`, {
    headers: { 'hibp-api-key': process.env.HIBP_API_KEY }
  });
  return resp.ok;
}
```

### Сложность внедрения

**Низкая сложность.** 4-6 часов.

### Аргументация

**Вердикт:** Внедрить. Простой API, высокая ценность для рисков.

# 3. НЕ ВНЕДРЯТЬ

Технологии не применимы к контексту INFOINDEXER.

---

## 3.1. SpiderFoot — НЕ ПОДХОДИТ ДЛЯ РФ

### Концепция

SpiderFoot — open-source фреймворк для автоматизированной OSINT-разведки с 200+ модулями.

### Почему НЕ внедрять

#### 1. Географическая специфика

SpiderFoot создан для западного рынка. Большинство модулей ориентированы на:
- Reddit, Tumblr, Flickr, Pinterest, Instagram
- WHOIS/SSL (полезно, но не приоритет)
- Западные соцсети и сервисы

**Для РФ нужны:**
- ВКонтакте, Одноклассники, Telegram
- Госзакупки, ФНС, ФССП
- Яндекс.Карты, 2ГИС

Этого в SpiderFoot нет.

#### 2. Уже есть лучшая альтернатива

INFOINDEXER уже реализовал собственный OSINT Waterfall с 21 источником, **специализированным для РФ**:

```
✅ ВКонтакте (два метода)
✅ LinkedIn
✅ Telegram
✅ Госзакупки
✅ Судебные реестры
✅ ЕФРСБ
✅ Росаккредитация
✅ HH.ru
✅ И другие...
```

SpiderFoot не покрывает большинство из этих источников.

#### 3. Инфраструктурные издержки

```
┌─────────────────────────────────────┐
│     INFOINDEXER (TypeScript)        │
└─────────────────────────────────────┘
                 │
        Требуется интеграция
                 │
┌─────────────────────────────────────┐
│      SpiderFoot (Python)            │
│  - Отдельный контейнер              │
│  - API integration                 │
│  - Data mapping                    │
└─────────────────────────────────────┘
```

- Ещё один контейнер в docker-compose
- Python-сервис в TypeScript-проекте
- API интеграция
- Фильтрация шумных данных

#### 4. Шум данных

SpiderFoot вернёт сотни артефактов, большинство из которых не релевантны для анализа российских компаний:
- Subdomain enumeration (зачем?)
- SSL certificate history (зачем?)
- IP blacklists (часто ложные срабатывания)

### Единственное исключение

**SpiderFoot имеет смысл ТОЛЬКО для:**
- WHOIS-lookup (если нужен инфраструктурный анализ)
- Email breaches (но проще напрямую через haveibeenpwned API)

### Вердикт

**НЕ ВНЕДРЯТЬ.**

**Альтернатива:** Расширять собственный OSINT Waterfall РФ-специфичными источниками (см. раздел 4).

## 3.2. Holehe — проверка email по сервисам

### Почему НЕ внедрять

1. Email директора есть редко (обычно только info@company.ru)
2. Этически сомнительная практика
3. Rate limiting со стороны сервисов

**Вердикт:** НЕ ВНЕДРЯТЬ.

---

## 3.3. Maigret / Sherlock — поиск по username

### Почему НЕ внедрять

1. Нет username в данных (только ФИО)
2. Дублирует существующий поиск по соцсетям

**Вердикт:** НЕ ВНЕДРЯТЬ.

---

## 3.4. Reddit-инструменты (Reveddit, RedditMetis)

### Почему НЕ внедрять

1. Reddit не популярен в РФ
2. Если нужен — добавить в existing sources

**Вердикт:** НЕ ВНЕДРЯТЬ.
