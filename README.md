# INFOINDEXER

Высокопроизводительная система сбора, индексации и многослойного обогащения данных о контрагентах РФ. Спроектирована для работы с Big Data объемами (30GB+) в условиях ограниченных вычислительных ресурсов.

## Архитектура

Проект реализован как модульный монорепозиторий (**NPM Workspaces**) с жестким разделением ответственности (**SoC — Separation of Concerns**). Это обеспечивает независимое масштабирование вычислительных воркеров и строгую консистентность типов между сервисами.

### Структура проекта

* \`apps/admin-ui\` — Аналитическая панель управления (Next.js, App Router).
* \`apps/sync-worker\` — Высоконагруженный ETL-процесс финансовых данных (Parquet/DuckDB).
* \`apps/egrul-sync-worker\` — Потоковая синхронизация государственных реестров (JSONL).
* \`apps/contacts-parser\` — Асинхронное обогащение данных (Playwright/OSINT).
* \`packages/shared\` — Core-пакет: Type-safe клиенты, общие доменные модели и утилиты.

## Слой данных и инфраструктура

Система оптимизирована для работы с OLAP-нагрузками и эффективного использования ресурсов:

* **ClickHouse** — основное аналитическое хранилище. Используются специализированные движки таблиц (MergeTree) для мгновенной фильтрации по выборке из **60млн+** записей.
* **Redis** — распределенный брокер задач и слой абстракции для кэширования, обеспечивающий **Observability** и отказоустойчивость воркеров.
* **DuckDB** — встраиваемое решение для локальной пре-агрегации данных. Позволяет обрабатывать Parquet-файлы «на лету» без избыточной десериализации в RAM, что критично для производительности ETL.

## Сервисы и пайплайны

| Сервис | Технологические особенности |
| :--- | :--- |
| **admin-ui** | Динамическая фильтрация (ОКВЭД, финансовые фильтры), визуализация трендов через Recharts, экспорт нормализованных данных. |
| **sync-worker** | Высокоэффективный стриминг данных. Реализована **Idempotency** (идемпотентность) загрузки с контролем целостности транзакций. |
| **egrul-sync-worker** | Обработка распределенных JSONL-дампов. Поддержка ротации SOCKS5-прокси для обеспечения непрерывности сбора данных. |
| **contacts-parser** | Автоматизированный OSINT-сбор через headless-браузеры. Внедрены алгоритмы **Deduplication** и верификации контактов. |

## Стек технологий

* **Runtime**: Node.js (TypeScript) — тотальная **Type-safety** всех DTO и внутренних интерфейсов.
* **Frontend**: Next.js 14, Tailwind CSS, shadcn/ui.
* **Database**: ClickHouse 24.x (LSM-tree storage), Redis 7.x.
* **Automation**: Playwright, DuckDB SDK.

## Быстрый старт

Система полностью контейнеризирована и готова к деплою через Docker Compose:

\`\`\`bash
# Подготовка окружения
cp .env.example .env

# Запуск всей инфраструктуры и приложений
docker compose up -d --build
\`\`\`

## Ключевой функционал

1. **Многомерный поиск** — высокоточная фильтрация по финансовым показателям, геопозиции и кодам классификации.
2. **Глубокая аналитика** — ретроспективный анализ выручки, активов и оценка рисков организации.
3. **Обогащение через OSINT** — автоматический поиск веб-сайтов, корпоративных почт и верификация данных о руководителях.
4. **Performance Optimization** — асинхронная обработка батчей данных и стриминговая передача результатов для минимизации задержек (Latency).
5. **Enterprise Security** — параметризация всех SQL-запросов, защита административных сессий и изоляция сетевого контура воркеров.
